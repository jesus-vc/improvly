# Design Lessons

**Personal pattern lessons discovered while building Improvly.** Not generic theory—real problems you solved and design decisions that clicked.

**Purpose**: Capture pattern lessons from your actual code with depth that would bloat AI instruction files.

## How to Use

- Reference these when making similar architectural decisions
- Keep AI instructions lean by storing detailed examples here

## How to Add New Pattern Lessons

**Only document patterns you personally discovered through coding.**

When adding:

1. **Show YOUR actual problem** - Use your real table/class/function that motivated the pattern
2. **Include YOUR reasoning** - What made you realize the original approach was fragile?
3. **Demonstrate YOUR solution** - Use your actual refactored code, not textbook examples
4. **Keep it brief** - Target 5-10 minute read time; concrete examples over theory

**Why personal examples matter**: Your brain remembers your own stories, not abstract principles. When you see `chain_labels`, you instantly recall the "favorite/archived/flagged boolean hell" you escaped.
